🗺️ Activity 1: Treasure Hunt ⏱ 8 min

You are an explorer. You've arrived at a mysterious island. Somewhere here, a treasure is hidden. Your only tool? The terminal.

Nobody is going to tell you where the treasure is. You have to find it yourself.

Getting Started

Open your terminal and go to the treasure hunt:

cd ~/linux-adventure/treasure-hunt

Now look around. What do you see?

ls

Your adventure begins with start-here.txt. Read it:

cat start-here.txt

Your Mission

  1. Read the first clue
  2. Follow the clues from folder to folder
  3. Find the treasure

Rules

  • Read every file you find — some are clues, some are traps!
  • If you get lost, type pwd to see where you are
  • To go back, type cd ..
  • The treasure might be hidden... 👀

Commands You'll Need

CommandWhat It Does
lsSee what's in the current folder
ls -aSee EVERYTHING — even hidden files!
cd foldernameGo into a folder
cd ..Go back one folder
cat filenameRead a file
pwdSee where you are right now
Start exploring. Read the clues. Follow the trail.
Can you find the treasure? 🏆
🤖 Activity 2: Robot Factory ⏱ 8 min

Welcome to the Robot Factory! Your job: build a robot from parts. You'll create robot parts as files, name them, and show your robot off in the showroom.

Getting Started

Go to the Robot Factory:

cd ~/linux-adventure/robot-factory
ls

Check what parts are available:

ls parts-bin/

Challenge 1: Build Your Robot

Go to the assembly line and create your robot's parts:

cd assembly-line
touch head.txt body.txt left-arm.txt right-arm.txt left-leg.txt right-leg.txt

Check your work and count the parts:

ls
ls | wc -l

Challenge 2: Name Your Parts

Rename your robot parts — get creative!

mv head.txt super-brain-3000.txt
mv body.txt titanium-chest.txt

Rename the rest however you want! Then check with ls.

Challenge 3: Move to the Showroom

Your robot is ready! Move it to the showroom:

mv *.txt ../showroom/
cd ../showroom
ls

Your robot is on display! 🤖🎉

⭐ Bonus Challenges

  • MEGA Robot: Build a robot with 15 parts!
  • Name it: echo "THUNDERBOT-9000" > robot-name.txt
  • Copy a part: cp left-arm.txt spare-arm.txt

📝 Activity 4: First Script ⏱ 7 min

A script is a file that tells the computer what to do, step by step. When you write a script, you are programming.

Step 1: Go to the Script Lab

cd ~/linux-adventure/my-scripts

Step 2: Create Your Script

nano hello.sh

A text editor opens in your terminal!

Step 3: Type This Code

#!/bin/bash
echo "=============================="
echo "  Hello! I am a Linux script!"
echo "  Made by: [YOUR NAME]"
echo "=============================="
echo ""
echo "Today is:"
date
echo ""
echo "You are logged in as:"
whoami
echo ""
echo "🐧 Script complete!"

Replace [YOUR NAME] with your real name!

Step 4: Save and Exit nano

ActionKeys
Save the fileCtrl + O, then press Enter
Exit nanoCtrl + X

Step 5: Run Your Script!

bash hello.sh
🎉 YOUR SCRIPT JUST RAN!
You are officially a programmer!

⭐ Bonus: Make It Interactive

Open your script again (nano hello.sh) and add:

echo "What is your name?"
read name
echo "Hello, $name! Welcome to Linux!"

Save, exit, and run with bash hello.sh.

The read command waits for you to type something!

🔍 Bonus: Command Detective ⏱ Extra

Someone stole the last cookie from the jar. There are witnesses and suspects. Your job: search through the evidence and figure out who did it.

Getting Started

cd ~/linux-adventure/detective
cat case-file.txt

Investigate

Read witness statements and suspect files:

cat evidence/witness1.txt
cat evidence/witness2.txt
cat suspects/alex.txt
cat suspects/sam.txt

Use grep — Your Secret Weapon

Instead of reading every file, search for clues!

grep "cookies" suspects/*
grep "kitchen" suspects/*
grep "chocolate" evidence/*
grep -r "cookie" ~/linux-adventure/detective/

Solve the Case!

  1. Who was in the kitchen?
  2. Who likes cookies?
  3. Who had chocolate on their shirt?
  4. Who stole the cookie?
💡 grep is one of the most powerful commands in Linux. Real engineers and security analysts use it every day!
