🚀 Before the Workshop — Deploy Scripts Do this first!

Run these scripts on every laptop before students arrive. Each script can be downloaded directly from GitHub and run in one command.

📦 Step 1: Main Setup Script — setup.sh

Creates all activity folders and challenge files in ~/linux-adventure/. Safe to run multiple times — it resets everything fresh.

https://raw.githubusercontent.com/jonathanbrenes/linux-workshop/main/setup.sh

One-liner to download & run:

curl -sL https://raw.githubusercontent.com/jonathanbrenes/linux-workshop/main/setup.sh | bash

Or download first, then run:

curl -sLO https://raw.githubusercontent.com/jonathanbrenes/linux-workshop/main/setup.sh
bash setup.sh

Verify it worked:

ls ~/linux-adventure/

You should see: treasure-hunt/ robot-factory/ detective/ playground/ my-scripts/

🎮 Step 2: Fun Tools Installer — setup-fun.sh

Installs cowsay, fortune, sl, cmatrix, and figlet for the Terminal Playground activity. Requires internet and sudo.

https://raw.githubusercontent.com/jonathanbrenes/linux-workshop/main/bonus/setup-fun.sh

One-liner to download & run:

curl -sL https://raw.githubusercontent.com/jonathanbrenes/linux-workshop/main/bonus/setup-fun.sh | sudo bash

Or download first, then run:

curl -sLO https://raw.githubusercontent.com/jonathanbrenes/linux-workshop/main/bonus/setup-fun.sh
sudo bash setup-fun.sh

Verify it worked:

cowsay "Ready to go!"
⚠️ No internet at the school? Run Step 2 (fun tools) at home/office on each laptop beforehand. Step 1 (setup.sh) doesn't need internet, but you'll need the file on a USB drive.

USB alternative: Copy both scripts to a USB drive, then from Ubuntu: 
bash /mnt/c/Users/USERNAME/Desktop/setup.sh
sudo bash /mnt/c/Users/USERNAME/Desktop/setup-fun.sh

🎤 Welcome Script 2 minutes

Keep it SHORT and energetic:

"Today, you're not just using a computer. You're going to control it. No mouse. No menus. Just you, the terminal, and your brain. Linux runs servers, phones, robots, even spaceships. And today — it runs YOUR adventure. Let's go!"

Then immediately:

  1. Make sure every group has Ubuntu open
  2. Have them type cd ~/linux-adventure and press Enter
  3. Start Activity 1!
💡 Energy tip: Don't over-explain. Kids learn by doing. Get them into Activity 1 as fast as possible.
📋 Activity-by-Activity Tips

🗺️ Treasure Hunt 8 min

Goal: Navigate folders and find hidden clues

Key commands: ls, ls -a, cd, cat

  • Big reveal: The treasure is a hidden file (.treasure) — they need ls -a to see it!
  • Don't give away the hidden file trick right away — let them struggle for 1–2 minutes
  • If a group is stuck after 2 minutes, hint: "Have you tried seeing ALL the files?"
  • There's also a hidden bonus: .secret-star in the forest

🤖 Robot Factory 8 min

Goal: Create files, rename them, count them

Key commands: touch, ls, mv, wc -l

  • Students get creative with robot part names — encourage it!
  • Challenge fast groups: "Can you build a robot with 15 parts?"
  • Remind them: blue = folders, white = files

🎮 Terminal Playground 5 min

Goal: Have FUN — see the terminal can be playful

Requires: cowsay, sl, fortune, cmatrix (from setup-fun.sh)

  • If tools aren't installed, students can still try echo, cal, date, whoami
  • cmatrix is a crowd favorite — let them enjoy it! Press q to quit
  • fortune | cowsay always gets laughs

📝 First Script 7 min

Goal: Write and run a bash script in nano

  • Key moment: When they run their script and it WORKS — celebrate it!
  • Walk them through nano: type → Ctrl+O to save → Enter → Ctrl+X to exit
  • Emphasize: "You just wrote a PROGRAM. You are now a programmer."
  • This is the most common place students need help — circulate actively

🔍 Command Detective Bonus

Goal: Use grep to solve a mystery

  • Only for groups that finish everything early
  • Answer: Alex stole the cookie (was in kitchen, likes cookies, had chocolate on shirt)
  • Great for curious/advanced students who want a real challenge
👥 Group & Energy Management

🔄 Rotation Rule

  • Every activity, a different person types
  • Other group members help by reading instructions and suggesting commands
  • No one should type for more than one activity in a row

⚡ Energy Management

SituationWhat to Do
Energy dropsSwitch to Terminal Playground — fun commands wake everyone up
Groups racing aheadGive them the Command Detective bonus
Groups falling behindSkip Terminal Playground → go straight to First Script
Noise getting loudTerminal excitement is GOOD! Only redirect if groups distract each other. Say: "Indoor hacker voices"

🎯 When to Help vs. When to Observe

SituationAction
Student is thinking and trying commands👀 Observe — let them figure it out
Stuck for 2+ minutes💬 Hint — don't give the answer, give a direction
Typed something wrong, got an error👍 Encourage — "Errors are normal! Read the message."
Accidentally broke something🔧 Help — re-run setup.sh
Excited and wants to explore🚀 Encourage — "Try it! See what happens!"
Afraid to type💪 Reassure — "You can't break anything. Try it!"
🛠️ Troubleshooting
ProblemSolution
Ubuntu won't openStart menu → search "Ubuntu"
"Permission denied"Try adding bash before the script name
"command not found" for fun toolsRun sudo bash setup-fun.sh first
Student deleted files by accidentRe-run bash setup.sh — it resets everything
nano is confusingWalk them through: type → Ctrl+O → Enter → Ctrl+X
Terminal looks frozenThey might be in a program — try Ctrl+C or q
"No such file or directory"Check spelling, use ls to see what's available
WSL not installed on a laptopOpen PowerShell as Admin: wsl --install -d Ubuntu
⚠️ Quick Reset: If anything goes wrong on a student's laptop, one command fixes it: 
curl -sL https://raw.githubusercontent.com/jonathanbrenes/linux-workshop/main/setup.sh | bash
This re-creates all folders and files from scratch. Students can continue where they left off.
